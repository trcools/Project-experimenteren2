- Technische documentatie in een `technical/` submap

## Bijdragen

Bij het toevoegen of wijzigen van documentatie:
1. Gebruik duidelijke en begrijpelijke taal
2. Voeg voorbeelden toe waar mogelijk
3. Houd de documentatie up-to-date met code wijzigingen
4. Gebruik Markdown voor consistentie
